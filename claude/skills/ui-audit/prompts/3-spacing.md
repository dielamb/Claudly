# Audit 3 — Whitespace & Spacing Pressure Test

You are pressure-testing the whitespace and spacing in each image.

## Task

Work through these four questions for each image:

### MACRO SPACING (sections, containers)
- Are section gaps large enough to signal a new zone, or do sections bleed together?
- Is there a consistent spatial rhythm (e.g., an 8pt grid) or does spacing feel ad hoc?

### MICRO SPACING (components, text, icons)
- Inside cards and components, is padding equal on all sides or does it look squeezed?
- Do icons have enough clearance from adjacent text?
- Are any elements touching or colliding?

### BREATHING ROOM
- Which elements need more isolation to feel important?
- Where is whitespace being filled out of fear instead of intention?

### PERCEIVED VALUE
- Would increasing padding in any area make the design feel more premium? Where?
- Are there dense areas that could be split across two sections instead of one?

## Rules

- Give specific pixel recommendations
- If the project uses spacing tokens, name them
- Rank fixes by impact

## Output format

```markdown
## Audit 3 — Whitespace & Spacing

### {image-filename-1}
- **MACRO:** verdict + fix
- **MICRO:** verdict + fix
- **BREATHING ROOM:** verdict + fix
- **PERCEIVED VALUE:** verdict + fix

### {image-filename-2}
(same structure)

### Cross-viewport / cross-image pattern
- Spacing failures that persist across all images
- Image-specific spacing failures
- Single highest-impact change
```

Keep under 1200 words. Concrete px values only. No cheerleading.
