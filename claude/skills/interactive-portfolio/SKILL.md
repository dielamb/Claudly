# Interactive Portfolio Designer

Portfolios that convert, not just display. Every element earns its place.

## The 30-Second Test

A visitor landing on your portfolio must immediately understand:
1. **Who you are** — name, role, specialty
2. **What you do** — the specific value you deliver
3. **Your best work** — 1-3 standout projects visible above fold
4. **How to contact you** — one clear CTA, always accessible

If any of these take more than 30 seconds to find → redesign.

## Architecture Options

**Single-page scroll** (recommended for designers/creatives)
- Hero → Work → About → Contact
- Fast to scan, works on mobile, good for 3-6 projects

**Multi-page** (for extensive project portfolios)
- Separate pages per case study
- Better for deep work, longer reading time
- Needs strong navigation

**Hybrid**
- Single page overview + expandable/linked case studies
- Best of both: scannable + depth available

## Project Showcase

**Lead with impact, not tasks:**
```
BAD:  "Built a dashboard for analytics"
GOOD: "Redesigned analytics dashboard → 40% reduction in time-to-insight"
```

**Case study structure:**
1. **Challenge** — what problem, for whom, what was at stake
2. **Process** — key decisions made and why (not every step)
3. **Solution** — what was built/designed
4. **Results** — measurable outcomes (%, time, money, users)
5. **Learnings** — shows self-awareness, optional but powerful

## For Designers
- Show thinking, not just final screens
- Include: brief, constraints, iterations, decisions
- 3-5 deep case studies > 15 shallow ones
- Mobile responsiveness is non-negotiable

## For Developers
- GitHub links with real repos (not tutorial clones)
- Live demos > screenshots
- Show: real problem solved, technical depth, code quality
- Include contribution to open source if applicable

## Anti-Patterns to Avoid

| Pattern | Problem |
|---------|---------|
| Generic template | No differentiation — looks like everyone else |
| Style-heavy, weak projects | Signals substance problem |
| Resume-as-website | Misses interactive storytelling potential |
| Outdated work | First impression = your current level |
| No contact CTA | Conversion killer |
| Over-engineered portfolio | Portfolio complexity should match work complexity |

## Hiring Manager Reality

- Average time on portfolio: **90 seconds**
- They scan, not read
- They look for: problem-solving evidence, relevant experience, craft
- One strong project remembered > five forgettable ones

## Structure Template

```
HERO
[Name] — [Role] — [Specialty]
[One-line value statement]
[View work] [Contact me]

FEATURED WORK (3 projects max above fold)
[Project card: title, 1-line impact, thumbnail]

ABOUT (short)
[2-3 sentences: who, what, why different]

CONTACT
[One primary method — email or form]
[Social links secondary]
```
